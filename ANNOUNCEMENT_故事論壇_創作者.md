# 故事論壇系統 - 創作者指南

故事論壇是一個**自由發布、訂閱制通知**的系統，讓你能更有效地與讀者互動。

## 核心特色

✨ **無格式限制** - 自由發布任何形式的故事內容
📢 **主動推播** - 你決定何時通知訂閱者
👥 **權限管理** - 可授權協作者協助管理
📊 **訂閱統計** - 即時查看訂閱人數

## 快速開始

### 1. 發布故事

1. 在故事論壇頻道創建討論串
2. Bot 會私訊詢問是否開啟訂閱功能
3. 選擇「是」即可在討論串中創建訂閱入口

**自動詢問選項說明：**
- **是** - 立即創建訂閱入口
- **暫不** - 跳過此次，下次仍會詢問
- **不再詢問** - 永久關閉自動詢問功能

### 2. 創建訂閱入口（手動）

如果一開始沒有創建，之後也可以手動創建：
```
/sf entry
```

訂閱入口提供三種訂閱選項：
- **Release** - 正式發布通知
- **Test** - 測試/預覽通知
- **關注作者** - 追蹤你的所有作品

### 3. 發送更新通知

當你更新故事時，複製更新訊息的連結，然後：
```
/sf notify <訊息連結> type:release
```

**通知類型：**
- `release` - 通知訂閱 Release 的讀者（預設）
- `test` - 通知訂閱 Test 的讀者

**範例：**
```
/sf notify https://discord.com/channels/123.../456.../789... type:release
```

系統會自動 @ 對應類型的所有訂閱者，並附上訊息連結方便讀者跳轉。

### 4. 管理重要訊息

**釘選訊息**
```
?pin
```
- 回覆想要釘選的訊息
- 然後輸入 `?pin`
- 用於章節目錄、角色介紹等重要內容

**取消釘選**
```
?unpin
```
- 回覆想要取消釘選的訊息
- 然後輸入 `?unpin`

### 5. 授權協作者

如果有共同作者或編輯，可以授權他們協助管理：

**新增權限**
```
/sf permissions add @使用者
```

**移除權限**
```
/sf permissions remove @使用者
```

**查看權限列表**
```
/sf permissions list
```

**注意事項：**
- 最多 5 位使用者可以管理單一討論串（包含你）
- 被授權者可以使用 `/sf notify`、`/sf entry`、`?pin`、`?unpin`

## 使用技巧

### 善用訂閱類型

**Release（正式發布）**
- 用於正式章節、完整內容
- 大多數讀者會訂閱此類型

**Test（測試/預覽）**
- 用於草稿、測試內容
- 可以徵求反饋意見
- 避免打擾只想追正式版的讀者

### 推薦工作流程

1. **章節完成** → 發布到討論串
2. **複製訊息連結** → 右鍵點擊訊息 → 複製訊息連結
3. **發送通知** → `/sf notify <連結> type:release`
4. **重要資訊** → 使用 `?pin` 釘選章節目錄等內容

### 測試章節流程

1. **草稿發布** → 標註「測試版」
2. **發送測試通知** → `/sf notify <連結> type:test`
3. **收集反饋** → 只有訂閱 Test 的讀者會收到通知
4. **修改完成** → 發送正式通知 `type:release`

## 常見問題

**Q: 如何獲得訊息連結？**
A: 右鍵點擊訊息 → 選擇「複製訊息連結」

**Q: 可以同時通知 Release 和 Test 訂閱者嗎？**
A: 需要分別發送兩次通知，一次 `type:release`，一次 `type:test`

**Q: 發送通知時會 @ 到我自己嗎？**
A: 不會，系統會自動排除作者本人

**Q: 如果刪除訂閱入口會怎樣？**
A: 讀者仍可透過 `/sf subscribe` 指令訂閱，但建議保留訂閱入口方便新讀者

**Q: 可以查看有誰訂閱嗎？**
A: 目前只能看到訂閱人數統計，無法查看具體名單

**Q: 如何關閉自動詢問功能？**
A: 在 Bot 詢問時選擇「不再詢問」選項

## 最佳實踐建議

1. **在討論串開頭創建訂閱入口**，方便讀者發現
2. **善用 Test 類型**徵求草稿反饋，避免打擾只追正式版的讀者
3. **釘選章節目錄**等重要資訊，提升閱讀體驗
4. **授權可信賴的協作者**，分擔管理工作
5. **定期推送通知**，保持讀者的參與度
