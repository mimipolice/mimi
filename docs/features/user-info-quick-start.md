# User-Info 快速開始指南

## 🚀 5 分鐘上手

### 基本使用

```
/user-info @使用者
```

然後使用下拉選單切換不同頁面。

---

## 📊 6 個分析頁面

### 1. 📊 綜合資訊
**用途**: 快速了解使用者基本情況

**查看內容**:
- 使用者標籤、ID、建立時間
- 最活躍的伺服器 Top 10
- 最常用指令 Top 10

**適合場景**: 初步了解使用者

---

### 2. 💰 財務總覽
**用途**: 分析使用者的財務狀況

**查看內容**:
- 帳戶餘額（油幣、油票）
- 交易統計（轉入、轉出、淨收入）
- 支出/收入分類（8 大類別）
- 股票投資組合

**適合場景**: 
- 檢查異常財務活動
- 追蹤大額交易
- 分析收支來源

**關鍵指標**:
- 淨收入異常高 → 可能收入來源不明
- 特定類別支出過高 → 可能刷特定功能
- 股票持倉異常 → 可能操縱市場

---

### 3. 🤝 互動排行
**用途**: 找出主要交易對象

**查看內容**:
- 最常轉帳給您的人 Top 10
- 您最常轉帳的人 Top 10
- 可切換排序（💰 金額 / 🔢 次數）

**適合場景**:
- 找出關聯帳號
- 檢測互刷行為
- 追蹤資金流向

**使用技巧**:
1. 先按金額排序 → 找大額交易對象
2. 再按次數排序 → 找頻繁交易對象
3. 交叉比對 → 確認可疑關係

---

### 4. 🔍 使用模式分析
**用途**: 檢測異常使用行為（機器人、小帳）

**查看內容**:
- 最近 60 分鐘使用頻率
- 每個指令的詳細統計
- 可疑程度自動判定（✅ 正常 / ⚠️ 可疑 / 🚨 高度可疑）

**關鍵指標**:
- **使用間隔** - 太規律 = 可能是腳本
- **間隔穩定度 (CV)** - <10% = 極度規律 ⚠️
- **使用頻率** - 平均間隔 <2 秒 = 可能自動化
- **使用次數** - >200 次 = 異常多

**適合場景**:
- 檢測機器人
- 識別小帳
- 發現刷指令行為

---

### 5. 🕸️ 關係網路分析 ⭐ 最強大
**用途**: 深度分析帳號關聯，檢測小帳集團

**查看內容**:
1. **👑 PageRank 關鍵節點** - 找主帳號
2. **🏘️ Louvain 社群檢測** - 自動發現群組
3. **🔄 循環交易檢測** - 發現洗錢模式
4. **🚨 規則式集群** - 快速篩選
5. **🔗 直接關係** - Top 10 交易對象
6. **🔗🔗 間接關係** - 二度關係

**適合場景**:
- 稽查小帳集團
- 追蹤洗錢網路
- 發現互刷群組

**使用流程**:
```
1. 查看 PageRank → 找出關鍵節點（可能是主帳號）
2. 查看 Louvain 社群 → 自動發現緊密群組
3. 查看循環交易 → 發現洗錢模式
4. 查看直接關係 → 確認關係強度
5. 對可疑帳號逐一執行 user-info
```

---

### 6. 📝 詳細記錄
**用途**: 查看具體交易記錄

**查看內容**:
- 最近交易紀錄（15 筆）
- 卡片收藏總覽

**適合場景**: 查看具體交易細節

## 🎯 常見稽查場景

### 場景 1: 懷疑某人使用小帳
```
步驟：
1. /user-info @可疑帳號
2. 切換到「🤝 互動排行」
3. 查看最常互動的帳號
4. 切換到「🕸️ 關係網路分析」
5. 查看 Louvain 社群檢測
6. 如果發現可疑度 >70 的社群 → 確認為小帳集團

判斷標準：
- 社群模組度 >80% = 內部連接極度緊密
- 社群大小 3-10 人 = 典型小帳數量
- 外部連接 <20% = 封閉群組
```

### 場景 2: 檢測機器人刷指令
```
步驟：
1. /user-info @可疑帳號
2. 切換到「🔍 使用模式分析」
3. 查看指令使用統計

判斷標準：
- 間隔穩定度 CV <10% = 極度規律 🚨
- 平均間隔 <2 秒 = 可能自動化 🚨
- 使用次數 >200 = 異常多 ⚠️
- 可疑程度 = 高度可疑 🚨
```

### 場景 3: 追蹤洗錢行為
```
步驟：
1. /user-info @可疑帳號
2. 切換到「💰 財務總覽」
3. 查看異常大額交易
4. 切換到「🕸️ 關係網路分析」
5. 查看循環交易檢測

判斷標準：
- 發現循環交易（A→B→C→A）
- 循環總金額 >10 萬
- 循環次數 >50
- 可疑度 = 100/100 🚨
```

### 場景 4: 找出小帳集團主帳號
```
步驟：
1. /user-info @任一小帳
2. 切換到「🕸️ 關係網路分析」
3. 查看 PageRank 關鍵節點
4. 排名第 1 的通常是主帳號
5. 對該帳號執行 user-info 驗證

判斷標準：
- PageRank 分數明顯高於其他帳號
- 與多個小帳有高頻交易
- 財務總覽顯示大量轉出
```

## 💡 進階技巧

### 技巧 1: 批次稽查
```
1. 對可疑帳號執行 user-info
2. 記錄關係網路中的所有帳號
3. 對每個帳號執行 user-info
4. 比對使用模式和交易時間
5. 繪製關係圖
```

### 技巧 2: 交叉驗證
```
1. 使用模式分析 → 找出可疑帳號
2. 關係網路分析 → 找出關聯帳號
3. 財務總覽 → 確認資金流向
4. 互動排行 → 確認交易對象
5. 多維度驗證 → 提高準確度
```

### 技巧 3: 時間線分析
```
1. 查看「首次使用」和「最後使用」時間
2. 比對多個帳號的時間線
3. 如果時間高度重疊 → 可能同一人操作
4. 查看「最近 60 分鐘使用頻率」
5. 如果多個帳號同時活躍 → 高度可疑
```

## 🚨 可疑度評分參考

### 使用模式分析
- **0-2 分**: ✅ 正常 - 使用模式符合人類行為
- **3-4 分**: ⚠️ 可疑 - 有一些異常但不嚴重
- **5+ 分**: 🚨 高度可疑 - 強烈建議深入調查

### 關係網路分析

#### Louvain 社群
- **0-50 分**: ✅ 正常社群
- **50-70 分**: ⚠️ 需要注意
- **70+ 分**: 🚨 高度可疑，建議立即處理

#### 循環交易
- **0-60 分**: ⚠️ 可疑
- **60-80 分**: 🚨 高度可疑
- **80+ 分**: 🚨 極度可疑，幾乎確定違規

#### PageRank
- **>15%**: 👑 網路中的關鍵節點
- **10-15%**: ⭐ 重要節點
- **<10%**: 🟢 一般節點

## 🔧 故障排除

### 問題 1: 關係網路分析載入很慢
**原因**: 網路節點太多，計算量大
**解決**: 等待 3-5 秒，這是正常的

### 問題 2: 某些資料顯示「無紀錄」
**原因**: 該使用者沒有相關活動
**解決**: 這是正常的，表示沒有資料

### 問題 3: 可疑度分數看起來不準確
**原因**: 演算法閾值可能需要調整
**解決**: 參考 `docs/features/advanced-graph-analysis.md` 調整參數
