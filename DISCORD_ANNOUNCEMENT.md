# 📚 故事論壇系統更新公告

## ✨ 重大變更

**取消格式審查** - 故事論壇不再進行格式檢查，作者可以自由發布任何形式的內容！

系統現在專注於**訂閱與通知功能**，讓創作者與讀者更有效地互動。

---

## 👥 功能總覽

### 🎨 創作者功能

**自動詢問機制**
- 發布新討論串時，Bot 會詢問是否開啟訂閱功能
- 可選擇「是」、「暫不」或「不再詢問」

**訂閱入口**
- `/sf entry` - 在討論串中創建訂閱入口
- 提供三種訂閱選項供讀者選擇

**更新通知**
- `/sf notify <訊息連結> [type:release|test]` - 推播更新給訂閱者
- Release：正式章節通知
- Test：測試/草稿通知

**訊息管理**
- `?pin <訊息連結>` - 釘選重要訊息
- `?unpin <訊息連結>` - 取消釘選

**權限管理**
- `/sf permissions add <使用者>` - 授權協作者
- `/sf permissions remove <使用者>` - 移除權限
- `/sf permissions list` - 查看權限列表
- 最多 5 位使用者可管理單一討論串

---

### 📖 讀者功能

**訂閱方式**
1. 點擊訂閱入口按鈕（推薦）
2. 使用 `/sf subscribe [type]` 指令

**訂閱類型**
- **Release** - 正式發布通知
- **Test** - 測試/預覽通知
- **關注作者** - 自動訂閱該作者的所有新故事

**其他指令**
- `/sf unsubscribe` - 取消訂閱
- `/sf view` - 查看所有訂閱列表

---

## 📋 完整指令列表

### Slash Commands (`/sf`)

| 指令 | 權限 | 說明 |
|------|------|------|
| `/sf subscribe [type]` | 所有人 | 訂閱故事 |
| `/sf unsubscribe` | 所有人 | 取消訂閱 |
| `/sf view` | 所有人 | 查看訂閱列表 |
| `/sf notify <訊息連結> [type]` | 作者/授權者 | 發送更新通知 |
| `/sf entry` | 作者/授權者 | 創建訂閱入口 |
| `/sf permissions add <user>` | 作者 | 新增權限 |
| `/sf permissions remove <user>` | 作者 | 移除權限 |
| `/sf permissions list` | 作者 | 查看權限 |

### Prefix Commands

| 指令 | 權限 | 說明 |
|------|------|------|
| `?pin` | 作者/授權者 | 釘選訊息（回覆訊息後使用） |
| `?unpin` | 作者/授權者 | 取消釘選（回覆訊息後使用） |

## 🎯 核心特色

✅ **無格式限制** - 自由發布任何形式的故事
✅ **訂閱制通知** - 讀者主動訂閱，避免打擾
✅ **多種訂閱類型** - Release / Test / 關注作者
✅ **訊息連結推播** - 通知直接連結到更新內容
✅ **權限管理** - 可授權協作者協助管理
✅ **自動詢問** - 發文時可選擇是否開啟訂閱

---

## 💡 快速上手

**創作者：**
1. 發布討論串 → Bot 詢問是否開啟訂閱
2. 選擇「是」→ 自動創建訂閱入口
3. 更新故事後 → 複製訊息連結 → `/sf notify <連結>`

**讀者：**
1. 在故事討論串中找到訂閱入口
2. 點擊按鈕選擇訂閱類型
3. 作者更新時會自動收到 @ 通知

---

## ⚠️ 注意事項

- 每個討論串只能有一個訂閱入口
- 最多 5 位使用者可管理單一討論串（含作者）
- 作者發送通知時不會 @ 自己
- 訂閱資料會永久保存，需手動取消
- `/sf` 指令僅在故事論壇頻道的討論串中可用
- `?pin` 和 `?unpin` 需要回覆想要操作的訊息後使用
